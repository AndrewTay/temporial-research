01

Creating production-ready 3D models

Our upstream is Hunyuan3D-2 plus hand-authored base archetypes; the open per-upload gap is texture transfer (still a passthrough). These three target topology, riggable generation, and PBR texturing.

TOP 1

MatLat

CVPR 2026 · Highlight · KAIST

Material Latent Space for PBR texture generation — albedo / normal / roughness / metallic on a given mesh from text + geometry.

Directly attacks our #1 critical path: the texture-transfer stub. Code + weights released.
Flagged · toolchainarXiv ↗
TOP 2

Rigel3D

arXiv 2026 · 2605.13129

Rig-aware latents that jointly generate geometry + skeleton + skin weights, so assets are born rigged.

Bullseye for authoring our base archetypes already rigged — collapses generate+rig into one step.
No code yetarXiv ↗
TOP 3

Strips as Tokens (SATO)

SIGGRAPH 2026 · ACM TOG

Artist mesh generation with native UV segmentation — clean quad/tri edge-flow and ready-made UV charts.

Fixes the topology + UV loss from our decimation step, giving bake-ready charts for texturing.
Code coming soonarXiv ↗
Flagged · toolchain

MatLat — toolchain-blocked on this box

MatLat's inference render core is nvdiffrast (rasterize / interpolate / texture / antialias) which JIT-compiles CUDA, plus cvcuda for UV padding. This machine has no nvcc / MSVC and no CUDA Toolkit, so the renderer can't build — and per the A4D pilot, installing that toolchain was deliberately avoided. The authors' own published results (not ours) live on their project page — linked below to avoid rehosting their media — and illustrate the capability we'd gain once a compiler is available.

MatLat result clips are the authors' own media. View authors' results ↗
To unblock: install CUDA Toolkit 12.8 + MSVC Build Tools (~8–10 GB), then pip install nvdiffrast and shim cvcuda→cv2.inpaint. Watch 16 GB VRAM (SD3.5 + T5) — likely needs 8-bit / SD3.5-medium.
Foundation · TRELLIS (CVPR 2025 Spotlight, Microsoft — Structured 3D Latents): the SLAT image/text→3D model that the 2026 4D work (MORPHOS) extends — a production-grade alternative to Hunyuan3D-2 that also emits mesh + 3DGS + radiance. Code + weights are public and it fits 16 GB (fp16), but it compiles the same CUDA stack this box can't build (nvdiffrast, spconv, kaolin, flash-attn, diffoctreerast) — runnable only via Docker/WSL or a prebuilt-wheel Windows fork. repo ↗
Category 2
02

Animating 3D models

Three routes to growing an animation library automatically — rig+motion from a deforming mesh, text→animal motion, and 4D-from-video. All three are 2026 papers still awaiting code.

TOP 1

RigMo

arXiv 2026 · 2601.06378

Unifies rig + motion learning from raw mesh sequences (Gaussian bones + skin), with a Motion-DiT for generation. Works on quadrupeds (fox).

Could auto-discover rig and generate motion straight from a deforming mesh — a path past hand-keying.
Code coming soonarXiv ↗
TOP 2

OmniZoo / Topology-Agnostic Animal Motion

arXiv 2026 · 2512.10352

Text→animal motion across 140 species and arbitrary skeletal topology (OmniZoo dataset, 33k sequences).

A text-driven motion source that already spans cats & dogs — drop-in clip generation for our rig.
No code yetarXiv ↗
TOP 3

MORPHOS

arXiv 2026 · 2606.02491 · KAIST

Autoregressive 4D generation from video (Temporal Structured Latents) → dynamic mesh / 3DGS / radiance.

SOTA successor to the ActionMesh 4D path we piloted — motion straight from a driving video. Benchmarked on a B200, so heavy.
No code yetarXiv ↗
Category 3
03

Retargeting mocap onto 3D models

Retargeting binds to the rig + skin. These three frame the space — video→BVH retarget, the rig+skin model itself, and a quadruped pose manifold.

TOP 1

MoCapAnything

arXiv 2026 · 2512.10881

Monocular video + a rigged asset → a BVH that drives that asset; explicit cross-species / quadruped retarget.

Literally "retarget mocap onto 3D models" for animals — would auto-bind generated motion onto our rig.
Code · no weightsarXiv ↗
TOP 2 · BUILT

Skin Tokens / TokenRig

SIGGRAPH 2026 · 2602.04805

Unified autoregressive rigging + skinning via learned skin tokens; reports +98–133% skinning accuracy over prior.

Unified rig + skin in one pass — the substrate retargeting binds to; we run it fresh on cat & pug as the live Category-3 result.
Built ✓arXiv ↗
TOP 3

ViPS

arXiv 2026 · 2604.17623

Video-informed pose spaces for auto-rigged meshes — a species-agnostic pose manifold (127k poses, 100+ species).

Pose-space retargeting + editing for quadrupeds without artist-authored 4D data.
No code yetarXiv ↗
Built & tested

Ran the released model in free-gen, texture-preserving mode on both raw meshes. Each got a full skeleton + per-vertex skin in ~18–20 s on the 5070 Ti, with original textures and geometry preserved. Method & honesty notes
05

How this was built — and the caveats

Hardware: RTX 5070 Ti (16 GB, Blackwell sm_120), Ryzen 7 7800X3D, 31 GB RAM, Windows 11. torch 2.x + cu128. No CUDA/C++ toolchain (prebuilt wheels only).

TokenRig: ran the released model (free-gen, texture-preserving) on the raw cat/pug GLBs → 27/28-joint rigs, textures kept. Turntables rendered headless in Blender (EEVEE).

Caveats: the 2026 animal-motion papers are all code-gated (RigMo / OmniZoo "coming soon", MORPHOS a teaser), so Category 2 has no live build. MatLat is genuinely runnable but needs a compiler this box lacks, so its tiles are the authors' results. All "no-code" flags reflect repo state as of this scan (Jun 2026) and may change.
